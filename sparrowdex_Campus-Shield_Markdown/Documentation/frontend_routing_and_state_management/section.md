The Campus-Shield application utilizes React Router for navigation and React Context API for global state management. This architecture enables seamless navigation between different pages and components while maintaining a centralized state accessible throughout the application.

<artifact ArtifactUUID="559387e6-c10b-4a84-85d8-3dd5cdabc686">Routing Configuration and Main Application Structure</artifact>

The routing configuration is defined in `App.tsx`, which serves as the main entry point for the application:

```tsx
import { BrowserRouter as Router, Route, Routes } from 'react-router-dom';

function App() {
  return (
    <Router>
      <AuthProvider>
        <NotificationProvider>
          <Navbar />
          <Routes>
            <Route path="/" element={<Home />} />
            <Route path="/login" element={<Login />} />
            <Route path="/register" element={<Register />} />
            <Route path="/admin" element={<AdminDashboard />} />
            <Route path="/report" element={<ReportIncident />} />
            {/* Other routes */}
          </Routes>
        </NotificationProvider>
      </AuthProvider>
    </Router>
  );
}
```

This setup wraps the entire application in `Router`, `AuthProvider`, and `NotificationProvider` components, ensuring that routing and global state are available throughout the app.

```tsx

This setup allows for centralized notification management across the application.

const Navbar: React.FC = () => {

  return (
    <nav>
      {/* Navigation links based on user role */}

      {user?.role === 'user' && <Link to="/report">Report Incident</Link>}
      {/* Other navigation items */}
    </nav>
  );
};
```

Route protection is implemented using custom route components or hooks that check the user's authentication status and role before rendering protected pages.

### State Usage in Components

Components like `Home`, `Login`, `Register`, and `AdminDashboard` consume the global state:

```tsx
const Home: React.FC = () => {
  const { user } = useAuth();
  const { notifications } = useNotification();

  return (
    <div>
      {user ? <WelcomeMessage user={user} /> : <LoginPrompt />}
      <NotificationList notifications={notifications} />
      {/* Other home page content */}
    </div>
  );
};
```

This architecture allows for efficient state management and routing, enabling a responsive and dynamic user experience in the Campus-Shield application.
